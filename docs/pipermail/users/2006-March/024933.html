To whom it may concern:

Please do not distribute cfengine 2.1.19 as long as a bug with how
cfengine generates symbolic links has been taken care of.

With cfengine it is possible to distribute i.e. a client.ntp.conf and a
server.ntp.conf and then link ntp.conf to the correct file depending on
the machine being a time server or just a client.

That doesn't work anymore, which killed sshd on several of my servers
(and tells me to do better testing before pushing out packages myself).

Thanks,

Ralph
