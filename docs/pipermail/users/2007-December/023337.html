Hi All

 

There's a program called postfix-policyd-spf-perl that helps stop spam by
verifying the sender is allow to send mail from that server.

 

I've been using it for a few months on my low volume Debian mail server, but
have just switched to CentOS and wanted to package it up.

 

Please find my .spec file attached.  I don't have a lot of experience with
this, so you'll want to double check it first.  Please also let me know if
you have any suggestions.

 

I explicitly specified a dependency on Perl >= 5.6 and postfix, the Perl
modules seem to be autodetected correctly.

 

It also requires a change to the master.cf Postfix configuration file.  This
is already documented in the INSTALL file.
