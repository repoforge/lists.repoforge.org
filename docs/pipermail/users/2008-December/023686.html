Roelof van der Kleij wrote on Thu, 11 Dec 2008 12:21:17 +0100:

> So shouldn't at least one of these spf modules be installed by default? 
> Or does 'require' only refer to minimal spamassassin functionality and 
> not to all functionality enabled by the default settings in the rpm?

the "module not installed" is just informational. You will get a "not 
installed" for SPF::Query even if SPF is already installed. You have to 
make your own interpretation from it. However, if as you say SPF is 
enabled in init.pre by default then it makes sense to add it as a 
dependency for the SA rpm, yes.

The spec file from spamassassin.org has just this
Requires: perl >= 5.6.1 perl(HTML::Parser) perl(Digest::SHA1) perl
(Net::DNS)

Looking again at the requires of the rf package:
Requires: procmail, gnupg, perl(Net::DNS), perl(Time::HiRes), 
perl-libwww-perl
Requires: perl(Archive::Tar) >= 1.23, perl(IO::Zlib), perl
(IO::Socket::SSL)
Requires: perl(DB_File), perl(LWP::UserAgent), perl(HTTP::Date)
Requires: /sbin/chkconfig, /sbin/service

I think they are already too strict. e.g. procmail is definitely not a 
requirement for SA and chkconfig/service only if you are running spamd.
