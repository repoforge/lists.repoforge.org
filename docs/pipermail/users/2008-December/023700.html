But the EPEL repository is not compatible with RPMforge and thus not 
recommended to use together. If you want to use EPEL, then you have to get 
rid of some existing RPMforge package and disable the repository in 
yum/apt/smart.

So for the majority of the people on this list it is easier to use the 
rkhunter from RPMforge. And so it makes sense to request it to be updated.

As long as EPEL is incompatible with RPMforge I would prefer if the advice 
to use the one from EPEL is not provided on this list. We have already had 
people complain when they followed exactly your advice and I don't want 
neither the complexity, nor those problems for the RPMforge users.

I hope you understand this concern.

besides we are also not rcommending RPMforge packages on EPEL or Fedora 
mailinglists.
