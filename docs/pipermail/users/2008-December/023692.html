> So, if you don't have make and gcc installed Unixbench 5 won't compile and

> When I yum removed unixbench it didn't remove the dependencies it had 
> installed. Shouldn't it do that? At least for CentOS base packages I find

> unwanted as it could remove a lot of basic packages. Is that why the 
> package didn't want to remove the three graphics libraries?
>
> Kai
>   
No, I don't think they should remove all of those. Packages are often 
dependent on core system utilities, such as /bin/sh, /bin/hostname, 
/bin/rm, glibc, and others. If you remove all the packages a package 
*required*, you can get into a cascading mess where you wind up ripping 
out nearly everything.
