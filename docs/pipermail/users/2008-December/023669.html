Hi all,

I just installed spamassassin 3.2.5-1.el5.rf on a minimal Centos 5 
install. When running sa-update -D I noticed some required perl modules 
are missing (see below). Since they are not mentioned in the requires in 
the spec file I wondered if perhaps the spec file needs updating. 
Spamassassin will run without them (at least as an amavis module, which 
is what I am doing), but I assume not all rules will work.

Some missing modules are related to rules which are disabled by default 
(like for instance DKIM)
Some are related to active rules (like  SPF)
Some are related to active rules which need binaries which in turn pull 
in the required perl module (the razor check is enabled by default, but 
razor-agents which in turn pulls in perl-Razor-Agent is not a requirement)

I am new to packaging, so I don't know what the policy is for requires. 
If I can test anything, let me know.

Roelof
