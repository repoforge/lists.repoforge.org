> Ok no we get some fun:

> Any opions on how to solve that in a decent way?

Some things we cannot and should not fix, much as perl-XML-LibXML :-)

Another thing that might happen in the future (did happen in the past) is 
that we ship some package before Red Hat does (eg. dnsmasq) and then Red 
Hat adds it in a future update release but with an older version.

That means that people who had dnsmasq already installed, will not get Red 
Hat updates, since they have been using a newer version that predates Red 
Hat's package. But if we pull that package from the repository after the 
fact, they will also stop getting updates from us.

So there is no good solution either :-(
