Tried to install vlc on CentOS-5 i386 and ran into problems with portaudio:

# yum install vlc
...
vlc-0.9.9a-3.el5.rf.i386 from rpmforge has depsolving problems
   --> Missing Dependency: libportaudio.so.2 is needed by package 
vlc-0.9.9a-3.el5.rf.i386 (rpmforge)
Error: Missing Dependency: libportaudio.so.2 is needed by package 
vlc-0.9.9a-3.el5.rf.i386 (rpmforge)

Installing vlc on CentOS-5 x86_64 works.  On further investigation, the 
i386 portaudio version is portaudio-18.1-2.2.el5.rf while the x86_64 
version is portaudio-19-1.20071207.el5.rf

The i386 package only provides /usr/lib/libportaudio.so whereas the 
x86_64 package provides
/usr/lib64/libportaudio.so
/usr/lib64/libportaudio.so.2
/usr/lib64/libportaudio.so.2.0.0
/usr/lib64/libportaudiocpp.so
/usr/lib64/libportaudiocpp.so.0
/usr/lib64/libportaudiocpp.so.0.0.12
/usr/lib64/pkgconfig/portaudio-2.0.pc
/usr/lib64/pkgconfig/portaudiocpp.pc

The later i386 package appears at http://packages.sw.be/portaudio/
and I tried "yum clean all" and an update without success.  The package 
is installable directly with rpm - Seems like a metadata problem.
