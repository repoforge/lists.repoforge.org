This is a conflict between perl-DBD-mysql from rpmforge and
perl-DBD-MySQL from CentOS/RHEL

You should use protectbase or priorities plugin as described here:
http://wiki.centos.org/PackageManagement/Yum/Priorities
http://wiki.centos.org/PackageManagement/Yum/ProtectBase

CentOS/RHEL depends on perl-DBD-MySQL (note the capital letters) wich is
not provided by the rpmforge package. You can see the difference if you
try yum provides perl-DBD-mysql vs yum provides perl-DBD-MySQL
