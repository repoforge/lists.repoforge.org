On Tue, 2006-11-07 at 11:36 -0600, Ross Marshall wrote:
> I am running CentOS 3.8 i386 and needed to upgrade to python 2.4. After 
> doing so, I found alot of packages broken because 2.4 from source 
> doesn't include all of the modules. Is there anyway to gracefully 
> upgrade python? Is there a complete rpm that I can install? Or is there 
> an easy way to have the CentOS scripts use the 2.3 version, and the 
> other software use the 2.4?
> 
> Thanks for All your help, and good work..

I built back-port packages of 2.4.1 some time ago, but never really used
them, so I can only assume they work fine.  The changes between my spec
file and the stock spec file were minimal, though.  If you ever upgrade
to a later version of CentOS with a stock 2.4, you'll want to remember
to remove these, because the /usr/lib/python2.4 will conflict.

You can get the binary RPMs and spec file here:
http://ftp.nakedape.cc/pub/nakedape/rpms/backports/el3/rpm/python/

Or use the repo directly in Yum:
http://ftp.nakedape.cc/pub/nakedape/rpms/backports/el3/
