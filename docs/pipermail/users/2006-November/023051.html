You should use the JPackage.org specifications for building the JDK RPMs.
They already have high quality JVM RPMs built from binaries from Sun, IBM
and Bea.

What you have to do, in my opinion, if you want to take this job, is to look
how their final RPMs distribute the JVM files in the filesystem, understand
the JPackage.org standard and write new specs that do the job from the
source.

BTW, all Java-related software distributed in Fedora, RHEL, SUSE Linux,
Mandriva, etc use the JPackage standards. It all starts with a package
called jpackage-utils, that you already have installed on your system. BTW,
I stopped to use Java packages from RPMForge because they violate the
JPackage standard.

Avi
