see the recent posts by dag on this list, concerning the split of 
rpmforge into several repos. The rpmforge repo is now intended to never 
replace base packages, such packages are now in the new rpmforge extras 
repo (repo tag rfx). This includes perl-XML-SAX on EL4.
Briefly, you should configure yum to use these repos, using priorities 
and/or disabling rfx by default (or whatever your policy is regarding 
superseding base SL packages).
BTW you shouldn't use the "dag" repo, although I think it's a link to 
rpmforge currently.

 From what I understand base+rf should be "closed", so your report may 
mean that perl-XML-Simple should move to rfx in el4.
