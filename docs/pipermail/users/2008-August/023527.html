Here's an updated version of the python-genshi spec file that doesn't use the 
autogenerated filelist, and includes the egg.info for setuptools compatibility.

This is not a noarch package, as it compiles an accelerator from C code. This 
also means it wants to install into the platform-specific directory. I took a 
guess, and used:

%define python_sitelib %(%{__python} -c 'from distutils import sysconfig; 
print sysconfig.get_python_lib(plat_specific = True)')

Let me know if there's a better way to handle this.
