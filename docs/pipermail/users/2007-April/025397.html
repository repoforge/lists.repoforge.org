Morning all,

just some small request: Can jigdo in rpmforge be updated to version
0.7.3? 

0.7.1 won't build on RHEL 5 because of a) a dropped w3c-libwww package
and b) because of some error while building the package itself.

0.7.3 dropped w3c-libwww and uses libcurl for getting files.
So this will build on RHEL5 (I have no idea if libcurl in RHEL3 and
downwards is new enough for that ...)

Here's the patch:

--- jigdo.spec.org      2006-01-06 22:21:07.000000000 +0100
+++ jigdo.spec  2007-04-18 10:41:51.000000000 +0200
@@ -6,7 +6,7 @@
 
 Summary: Tool for distributing large files
 Name: jigdo
-Version: 0.7.1
+Version: 0.7.3
 Release: 1.rf
 License: GPL
 Group: Applications/Internet
@@ -18,8 +18,10 @@
 Source: http://atterer.net/jigdo/jigdo-%{version}.tar.bz2
 BuildRoot: %{_tmppath}/%{name}-%{version}-%{release}-root
 
-BuildRequires: w3c-libwww-devel, gtk2-devel, pkgconfig, db4-devel 
+BuildRequires: gtk2-devel, pkgconfig, db4-devel 
 BuildRequires: gcc-c++, openssl-devel, desktop-file-utils, wget
+BuildRequires: curl-devel, bzip2-devel
+
 Requires: wget
 %{?fc4:BuildRequires: compat-gcc-32, compat-gcc-32-c++}
