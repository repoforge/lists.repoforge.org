hello

I wanted uudeview in centos5, because it is required by nzbperl 
(http://noisybox.net/computers/nzbperl/ , for downloading binary usenet 
stuff from NZB files, looks pretty good BTW)

rpmforge has uudeview, but the build failed on centos5.

Actually the problem is that uudeview uses latex to generate 
documentation, and the latex file tries to \usepackage{psfig} which is 
not available in centos5 (deprecated IIRC).
However, the latex file doesn't seem to actually *use* anything from 
psfig.sty. Commenting the \usepackage{psfig} command in doc/library.ltx 
allows to build the package, and the documentation looks OK.

In case you want to include this in the rpm and my description above is 
confusing, here's the one-char patch


--- uudeview-0.5.20/doc/library.ltx.orig        2007-11-27 
21:23:05.000000000 +0100
+++ uudeview-0.5.20/doc/library.ltx     2007-11-27 21:23:11.000000000 +0100
@@ -1,6 +1,6 @@
  \NeedsTeXFormat{LaTeX2e}
  \documentclass{article}
-\usepackage{psfig}
+%\usepackage{psfig}
  \usepackage{times}
  \usepackage{a4wide}
  \author{Frank Pilhofer}
