On Mon, 25 May 2009, Nicolas Thierry-Mieg wrote:

> today apt offered a new version of the proprietary nvidia drivers from 
> rpmforge. However in addition to upgrading the two installed packages, it 
> also installed a new dkms-nvidia-x11-drv package.
> I used to have:
> nvidia-x11-drv-173.08-1.beta.el5.rf.x86_64
> nvidia-x11-drv-32bit-173.08-1.beta.el5.rf.x86_64
> I now have:
> dkms-nvidia-x11-drv-180.51-1.nodist.rf.x86_64
> nvidia-x11-drv-180.51-1.el5.rf.x86_64
> nvidia-x11-drv-32bit-180.51-1.el5.rf.x86_64
>
> AFAICS the two packages dkms-nvidia-x11-drv and nvidia-x11-drv seem almost 
> identical, except for a few cosmetic changes (dir names, PACKAGE_VERSION 
> value in dkms.conf, and /usr/share/applications/*.desktop file name change).
>
> I can also see that dkms-nvidia-x11-drv-180.51-1.nodist.rf.x86_64 obsoletes 
> nvidia-x11-drv <= 173.08 .
>
> I guess I should install dkms-nvidia-x11-drv as well as the (currently not 
> installed) dkms-nvidia-x11-drv-32bit, and uninstall nvidia-x11-drv and 
> nvidia-x11-drv-32bit ?
>
> If this is true, probably the dkms-* packages should obsolete all versions of 
> nvidia-x11-drv* ?
>
> Is this correct?

Yes, I don't know why it happened, by accident the nvidia-x11-drv 
x86_64 package was also released, while in fact only the 
dkms-nvidia-x11-drv package should have been released.

The aim was to move from nvidia-x11-drv to dkms-nvidia-x11-drv so that the 
twin kmod-nvidia and nvidia-x11-drv package from elrepo would not 
conflict.

So if you see an nvidia-x11-drv package, please remove it.

I already change the repositories, but that will only have effect tomorrow
