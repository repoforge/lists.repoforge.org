I've not tried 'mach'. But may I recommend the 'mock' package,
available as part of CentOS and with newer versions from both EPEL and
RPMforge? Download a local mirror of the distribution of the OS you
want to build such chroots for: it's far faster to get them from a
local mirror for new builds than from some remote location.

Do you have any experience for mach versus mock? I admit that the
'/etc/mach/location' file looks easier to manage than the set of
/etc/mock/ files to point to new repositories. mach also doesnt seem
to point to configuration tools outside itself, such as mock pointing
to 'http://buildsys.fedoraproject.org/'
