On Fri, 2008-11-07 at 11:40 -0800, MargoAndTodd wrote:

> Now over on Xubuntu <8.10 side there is a utility

Why do you think this particular package has to do something with your
problem? Check out

http://packages.ubuntu.com/intrepid/firefox-gnome-support

It's a meta-package which points to the latest gnome-support stuff. It
contains:

/usr/share/doc/firefox-gnome-support/changelog.Debian.gz
/usr/share/doc/firefox-gnome-support/copyright

Moreover, it gets generated while compiling firefox-3.0 package so you'd
better look through the patches in the firefox-3.0_3.0.3
+nobinonly-0ubuntu2.diff.gz pack. I haven't found anything remotely
useful though.

However after some research I have found an interesting bug @ Launchpad:

https://bugs.launchpad.net/archlinux/+bug/35997

To make the long story short, I would try to look for

$ grep directory /etc/gnome/defaults.list
inode/directory=nautilus-folder-handler.desktop
x-directory/normal=nautilus-folder-handler.desktop

And change them to

Thunar-folder-handler.desktop

In any case it does not seem to have anything to do nor with RPMForge,
neither with firefox-gnome-support.
