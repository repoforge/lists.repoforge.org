Hi,

It seems the recent debate with wxGTK shows an issue that is frequently
recurring.

Take wxGTK 2.8 for example. If you need 2.6 then having 2.8 intalled is
not going to help you.

While this may not be possible for all packages at least some of these
issues could be solved if at the time there is a version jump a backward
compatibility package is created.

So when a wxGTK 2.8 package is released and there was a 2.6 package
before. Then a minimal set of files could be put into a compat-wxGTK26
package.

That might take care of dependencies. Or it might not. It is just a
thought I had that I have not been able to put to the test (yet).

While it is not very clean way of doing things it might solve at least
some dependencies. It also means more work for the packagers. But the
extra work might pay off.
