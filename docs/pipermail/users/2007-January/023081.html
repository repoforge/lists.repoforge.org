Should be as simple as rebuilding the src rpm, installing it, restarting
spamd, and run 'spamassassin --lint' to check for errors (should be no
output if ok).  

You can test an email from the command line with "spamassassin -t <
samplefile.eml" and you should along with all the other output, see
something like:

5.0 BOTNET                 Relay might be a spambot or virusbot
[botnet0.7,ip=217.226.209.237,hostname=pD9E2D1ED.dip.t-dialin.net,maildo
main=alexalvarez.com,client,ipinhostname,clientwords]

(From your other email)
> error: line 3: Tag takes single token only: Release: %smerelease 1

Sorry, that is a quirk with our build system.  Either edit the spec file
or add something like this to your .rpmmacros file so it will build:

%_packager Your Name <you at somewhere.com>
%smerelease() %(echo "%{?1}.el4.sme.%{?2}" | sed -e's|\\.$||')
