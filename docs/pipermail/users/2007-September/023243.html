One thing to fix in the spec file they provide in the tarball is the directory:

drwxr-xr-x  2 root root 4096 Sep 25 12:34 /var/milter-greylist/

it must be:

drwxr-xr-x  2 smmsp smmsp 4096 Sep 25 12:34 /var/milter-greylist/

as the /etc/mail/greylist.conf file runs milter-greylist as the user "smmsp". 

If the daemon doesn't have write permission to this directory it fails to run
and bombs out.
