Somewhat surprising as you typically want either baseurl or mirrorlist
but not both.  If baseurl is taking precedence, you are hitting the main
server and not taking advantage of the mirrorlist (or fastestmirror if
using that plugin).  I presume you are changing 4 to 5 for your CentOS-5
machines.  My definition, which works for both 4.5 and 5 is:

# Name: RPMforge RPM Repository for Red Hat Enterprise $releasever - dag
# URL: http://rpmforge.net/
[rpmforge]
name = Red Hat Enterprise $releasever for $basearch - RPMforge.net - dag
#baseurl = http://apt.sw.be/redhat/el$releasever/en/$basearch/dag
mirrorlist = http://apt.sw.be/redhat/el$releasever/en/mirrors-rpmforge
#mirrorlist = file:///etc/yum.repos.d/mirrors-rpmforge
enabled=1
protect=0
#gpgkey = file:///etc/pki/rpm-gpg/RPM-GPG-KEY-rpmforge-dag
gpgkey = http://apt.sw.be/rpmforge/persona/dag/RPM-GPG-KEY-rpmforge-dag
gpgcheck=1
priority=4

Unfortunately mirrorlist seems not to understand &releasever so if using
local mirrorlist files they are unique to the release.
