Andrew Ziem wrote :

> There seems to be a conflict in x264 between rpmforge and freshrpms.  
> After installing x264 and Kino from freshrpms, this error occurs on upgrade:
> 
> --> Running transaction check
> --> Processing Dependency: libx264.so.48 for package: kino
> --> Finished Dependency Resolution
> Error: Missing Dependency: libx264.so.48 is needed by package kino
> 
> The strange part is that "rpm -qR kino | grep 264" prints nothing, and I 
> can remove x264 without removing Kino.

x264 used to be only a static lib, so the kino you have must have it
statically linked in. Only recently was x264 built as a shared lib, and
since its version keeps changing quite fast, it's causing some issues.

Normally FC5 freshrpms should still have the static x264 IIRC, and FC6
freshrpms will start with the dynamic one. If there is anything I can
do in the freshrpms repository to help solve this, please let me
know :-)
