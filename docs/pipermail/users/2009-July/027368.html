Hi all,

Following on from this thread:
http://lists.rpmforge.net/pipermail/users/2009-July/002652.html

It seems that the rtorrent package for el5 is broken on RPMForge with
respect to connecting to trackers. It seems to be an issue with the version
of libcurl expected. When running this version, the system sits there
forever when trying to contact the tracker. Eventually it timeouts, waits
and tries again. And again. And again.

I found version 0.7.8 of rtorrent and 0.11.8 of libtorrent on EPEL which
seems to work - however no such luck with the builds from RPMforge.

The victim system is a CentOS 5.3 box with all updates applied.

$ rpm -qa | grep curl
curl-7.15.5-2.1.el5_3.4

and at the moment, these from EPEL:
$ rpm -qa | grep torrent
libtorrent-0.11.8-1.el5
rtorrent-0.7.8-1.el5

Thoughts? Ideas?
