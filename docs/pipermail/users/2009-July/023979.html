On Fri, 17 Jul 2009, Amos Shapira wrote:

> Would you please provide a package for perl Time::ParseDate
> (http://search.cpan.org/~muir/Time-modules-2006.0814/lib/Time/ParseDate.pm)
> for CentOS 5?

[root at lisse rpms]# /dar/tools/dar/dar-perl.py -c perl-Time-ParseDate
Module Time::ParseDate found in package Time-modules
Warning: License could not be determined.
Warning: No abstract found.

So the module in fact already exists as part of the perl-Time-modules 
package. The standardized way of packaging perl modules is using the perl 
CPAN distribution name. I know it is confusing, but you can do a lookup 
like this:

 	apt-cache whatprovides 'perl(Time::ParseDate)'
or
 	yum provides 'perl(Time::ParseDate)'
