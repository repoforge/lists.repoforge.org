I've just run into some problems caused by weird numbering with
some of the Perl packages in RPMForge in the el4 repo (at least).

The package that bit me was perl-Devel-StackTrace.  There are
packages with version numbers that are sorting badly (compare
versions with dates):

perl-Devel-StackTrace-1.11-1.2.el4.rf.noarch.rpm  30-Mar-2005 21:47      13K
perl-Devel-StackTrace-1.12-1.2.el4.rf.noarch.rpm  27-Nov-2005 01:18      14K
perl-Devel-StackTrace-1.13-1.el4.rf.noarch.rpm    22-Sep-2006 05:57      14K
perl-Devel-StackTrace-1.15-1.el4.rf.noarch.rpm    19-Jun-2007 05:12      22K
perl-Devel-StackTrace-1.16-1.el4.rf.noarch.rpm    21-Feb-2008 12:41      24K
perl-Devel-StackTrace-1.18-1.el4.rf.noarch.rpm    04-May-2008 18:09      26K
perl-Devel-StackTrace-1.21-1.el4.rf.noarch.rpm    07-Jul-2009 02:59      28K
perl-Devel-StackTrace-1.1901-1.el4.rf.noarch.rpm  22-Jun-2008 18:34      26K
perl-Devel-StackTrace-1.1902-1.el4.rf.noarch.rpm  07-Oct-2008 15:51      26K

RPM thinks that 1.1901 and 1.1902 are much larger numbers (and
thus later) than 1.21.

There are a bunch of other perl-* RPMs that have the same issues,
but this one broke my Mason install on my development machine.
