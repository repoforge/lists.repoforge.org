I am running CentOS 5.3.  I thought it was just going to be two packages...
perl-MP3-Tag and perl-Unix-Process... then today, it added 5 more.

When I go to look for the dependencies, I am lucky if I can find one for Fedora 11,
and even if I do find that one, it won't install because of other dependency problems.
As an example, I found perl-Exception-Class-1.29.1 ... but it was for PLD, not CentOS.
When I restrict the search to be for CentOS/RHEL, the most recent version I find is
1.24... from the rpmforge repository!  I can find the perl(IPC::System::Simple) dependency,
but it's for Fedora 11, and to install that one on CentOS 5.3 has all kinds of other
dependency issues that cannot be resolved easily.

PLEASE... when you build these updated packages and start requiring new stuff, don't
release them as part of a general release without their dependencies.
