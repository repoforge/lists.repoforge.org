Hi everybody,

I've tried to use the RPMForge repo built for el5 on CentOS 5 (beta) but
i encountered some problems :

- some packages exist in rpmforge that need other packages not existing
anymore in [base] : for example xmms-mp3 exist on RF but xmms in not
included anymore. So either xmms-mp3 should maybe be removed from RF or
either xmms could be integrated in RF (my proposal, like it is in FC
extras) ... or wait for EPEL to provide it ... but due to latest
discussions , i doubt mixing repositories is a good idea .. :o)

- some packages still want to update [base] , as it was before so using
the yum-protectbase is still a good idea to prevent overwriting of
[base] packages (an example is libevent-1.3b-1.el5.rf)

- there are a mismatch like for example xine : when i tried to yum
install xine , it complained after downloading all the files :
Transaction Check Error:
  file /usr/lib/xine/plugins/1.1.4/xineplug_inp_mms.so conflicts between
attempted installs of xine-lib-moles-1.1.4-1.el5.rf and
xine-lib-1.1.4-1.el5.rf

This is normal because those 2 rpms contain same files : 
#rpm -qilp xine-lib-1.1.4-1.el5.rf.i386.rpm |grep xineplug_inp_mms.so 
/usr/lib/xine/plugins/1.1.4/xineplug_inp_mms.so
#rpm -qilp xine-lib-moles-1.1.4-1.el5.rf.i386.rpm |grep
xineplug_inp_mms.so 
/usr/lib/xine/plugins/1.1.4/xineplug_inp_mms.so

Thanks.
