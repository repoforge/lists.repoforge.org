The problem is that rar is a binary. It has not been compiled by us.
As a result RPM gets its dependencies from the binaries and they include 
more (and newer) information than the system provides.

Two possibilities:

 + Remove the newer version so yum doesn't complain (consider apt if this 
   behaviour is bothering you, apt picks the most suitable candidate)
   I do not see this as a real problem even though yum fails on it.

 + Manually remove the wrong dependencies from the package, which is a 
   hard thing to do since that is very specific to the distribution.

The last option for me is the proper one.
