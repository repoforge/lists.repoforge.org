It is completely fine to simply give up when it is too hard to backport to 
RHEL4. The older RHEL4 (and RHEL5) becomes the harder popular stuff can be 
backported and the less time we have to spend trying it.

But it doesn't mean we stop trying to build the easy stuff for older RHEL 
releases either. So there's some middleground where we have a balance 
between the effort to do it and the importance of having it.

subversion is really important, and in this case it might be worthwhile to 
see with the subversion people whether there is an interest to make it 
work with somewhat older distributions.

In fact there need to be more awareness to supporting enterprise 
distributions. Often it boils down to using a slightly older python and 
not relying on the latest always.
