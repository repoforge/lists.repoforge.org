I recently upgraded to 0.8.4 out of the RPMForge repo.

Since the upgrade, I've experienced an inability of rTorrent to  
connect to any trackers.  "Tracker: [Timed out]" is all I ever get.

I noticed on the LibTorrent Known Issues page -- http://libtorrent.rakshasa.no/wiki/LibTorrentKnownIssues#Inabilitytoconnecttotrackersorannouncestimeout 
  --  something that sounds exactly like my problem:

Inability to connect to trackers or announces time out
Usually due to a buggy libcurl version. Since version 0.8.2, rtorrent  
uses more advanced features of libcurl. While these work for many  
users, in unknown circumstances (depending on system setup? trackers?)  
it may cause announces to fail consistently. In that case please  
update libcurl to version 7.19.2 or higher.

However, my system has:
curl.i386                                7.15.5-2.el5            
installed
and that seems to be the newest available in the various repos (e.g.  
CentOS, RPMForge, ATRPMS, EPEL).

I tracked down the Curl developers' download page -- http://curl.haxx.se/download.html 
  -- but there also is only an EL5 RPM for curl 7.15 -- not 7.19.2 or  
higher.

So, I'm wondering if anyone is successfully running rTorrent 0.8.4 out  
of RPMForge?  If so, does it need libcurl 7.19.2 or greater as  
suggested by my symptoms and the known issue?  If so, how did you get  
that version installed?  If not, anyone have any other suggestions  
about what could not be working with rTorrent on my system?  (BTW,  
connectivity is fine; e.g. bittorrent still functions in Azureus, just  
not the newest rTorrent.)

Thanks for any help.
