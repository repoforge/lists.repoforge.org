> Digium actively recommends people to install from source anyway, to get

I tried packaging it a couple years ago. There was a really bad spec file 
included in the tarball that showed no understanding of RPM packaging, but 
I found a pretty good one from a 3rd party and fed back some enhancements.

"Install from source" can be done with RPM: One just needs to build a 
source RPM. Nothing magic about that. The only value in building from 
source instead of using a binary RPM is if custom build options are needed 
for each installation.

I guess I need to see what the current state of upstream packaging is and 
then compare that to the RPMforge guidelines to see what needs to be 
tweaked to make a submission.
