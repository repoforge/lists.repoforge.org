Agreed.

I wonder what's the best way and location to publish them. In the past I 
pushed such files together with the push of packages, but that's 
absolutely not what we want right now.

Maybe twitter is not such a bad idea afterall, especially if it would 
regularly report if there's a change:

  - Revision number of my local tree

  - An overview of the recent build successes and failures (not
    specifically for what dist/arch, but just in general)

So a twitter message could simply state:

   Build tree at rev1234, no builds triggered

   Build tree at rev1235 triggered perl-ABC-XYZ, subversion, alpine

If the list is too long we can either chop it, or continue in a second 
tweet. Although on mass-builds it might become unusable. But what is most 
important is to know if something moved, right ?

An RSS feed with links to buildlogs would be a useful addition.
