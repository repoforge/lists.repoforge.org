On 01/08/2010 05:25 PM, Christoph Maser wrote:
> That propably means one spec file per commit, not wait until it is
> built.

Thats right, just keep doing commits - but try and keep it to one spec 
file per commit. And every commit will trigger a build request. The 
metadata being collected from the commit's is :

pkgname
commiter
BuildRequires
Distro and Arch's to build for
Repository to build for ( Test or Production )

So I guess its now easier to see why its preferred to have only 1 spec 
in the commit, then the build-id can be the commit-id itself ( and 
thereby make it a bit easier to track build -> source )
