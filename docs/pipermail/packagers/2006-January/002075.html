On Fri, 27 Jan 2006, Who Knows wrote:

> I have read thru the archives of this list and cannot find a reference to the
> dist tag requirements for rpmforge.
> 
> ex:
> 
> 1.fc1.rf
> 
> 2.fc4.rf
> 
> What are the meaning or significance of the leading numeral 1 or 2 in the
> example above?

It provides an (unsupported) upgrade path between EL and FC releases.

  0.el2 < 0.rh7 < 0.rh9 < 1.el3 < 1.fc1 < 1.fc3 < 2.el4 < 2.fc4 

Even though not supported by Red Hat, it helps to make sure when you move 
from eg. FC3 to EL4, the rpmforge packages will be EL4 eventually.

You often see the same done by Red Hat but only in numerical format. The 
downside of that is that it´s much harder to compare releases than in the 
scheme we use.
