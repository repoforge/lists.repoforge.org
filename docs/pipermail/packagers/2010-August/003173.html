Hi

I would like to see per target platform subdirectories in RPMRepo. That
way it is much easier to maintain different versions of a package for
different targets. Updates will only affect one platform often this will
avoid some problems (e.g. http://trac.rpmrepo.org/trac/RPMRepo/ticket/2)

What is the target platform list anyway? el3, el4, el5, el6, .....

Another thing I would to see on RPMRepo is a _new_ empty repo wich is
kept clean with repomange/repoclosure. I don't mean to throw away what
is already in rpmforge but maybe we can weed it out a bit by manually
transfering only packages worth it to a new RPMRepo repo.

Going that way would also be a simple transition stragtegy from svn to
whatever-SCM.
