Hi all,

While we were at FrOSCon, Chris, Didi and me have looked at the current 
trac status. Here is a small report of what was done and still needs to 
be done :

- Chris installed Trac previously and configured a svnsync process to 
import the whole svn tree into a local repo that Trac can use directly 
(so a real svn sync and not just a `svn checkout` followed by `svn 
update`) . He configured the sync to occur through a cron job.
- I just looked with Didi about applying a theme to that Trac instance. 
We've just uploaded a small rpmrepo logo (one that was proposed a 
*while* ago) and Didi started to package a ThemeManagerPlugin for Trac 
(a spec file will land in RPMforge svn in a short time, thanks Didi ;-) 
) . So we'll look to change the default trac theme soon (even if that's 
not a priority number one, but still good to have)
- Problem to be solved : browsing the actual rpmforge/rpms tree through 
the Trac soucebrowser really kills the machines (both the server and the 
client side where firefox just hangs for a while). I don't know what can 
be done to boost perfs or use caching but we started to investigate.

More to come ...
