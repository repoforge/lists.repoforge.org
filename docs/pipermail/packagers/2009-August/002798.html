Christophe,

I googled for the discussion we had about this:

 	http://lists.centos.org/pipermail/centos/2006-June/023053.html

There might be others where this was discussed though. The reasoning is 
that perl-CPANPLUS should not be required if we have all perl modules 
packaged. If people require perl-CPANPLUS for something they need it to 
download from Internet and in that case they might as well install 
perl-CPANPLUS from there.

We don't want people to think they are using RPMs for everything but in 
the background things are being taken from the Internet automatically.
