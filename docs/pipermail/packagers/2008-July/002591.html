> Today I installed librsync-0.9.7-2.el5.rf.x86_64.rpm and
> librsync-devel-0.9.7-2.el5.rf.x86_64.rpm, but I found the librsync.so was
> missed when I installed another package(csync2) which depends librsync.so.1
> dynamically.  The reason is the default configure script of librsync does
> not enable shared library building,  and the so files do not be included
> into the final rpm package.

Ok, let me give you the explanation to that. Almost in all cases where 
configure does not automatically create a shared library, the developers 
do not want to have a shared library.

Mostly because the interface is not deemed stable, and they are not 
versioning their libraries yet. In those cases it may be better to not 
have shared libraries at all.

If you think this is a mistake and not the intention of the librsync 
developers, I would appreciate if you contact them and ask them. I am 
willing to provide a shared library only if that is the intention of 
upstream.

> repository saved me a lot of time. Thank you.
