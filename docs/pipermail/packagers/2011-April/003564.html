On Mon, 2011-04-11 at 16:38 +0200, Dag Wieers wrote:

> Already imported dstat, mrepo and unoconv into my own github repositories. 

FYI, instead of GitHub importer you can use my import script (which uses
git-svn) with the latest version of the author list. If you don't need
branches / tags, just remove the corresponding options to "git svn".

You might need to make minor cleanups in the .git folder, but the upside
is that the author names and e-mails will be set nicely and correctly. 

I am not sure if it matters too much for your personal projects, but I
love browsing the history through gitk and nice name / e-mail pairs are
so good for the karma :-)
