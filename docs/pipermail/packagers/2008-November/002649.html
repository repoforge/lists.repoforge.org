On Fri, 21 Nov 2008, Dries Verachtert wrote:

> On Thursday 06 November 2008 09:46:49 pm Christoph Maser wrote:
>> Hi
>>
>> i am currently working on a ndoutils package. I would like to add a self
>> made init-script and logrotate config. I would like to avoid to put them
>> inline in the spec-file with cat << EOF ....
>>
>> What is the rpmforge way of providing my additional sources?
>
> Most packages contain those small files inline, see for example the
> dovecot.spec file. They use the 'EOF' approach.
>
> Other packages use separate files which are checked in the svn repository in
> the same directory, for example:
> http://svn.rpmforge.net/svn/trunk/rpms/collectl/
> http://svn.rpmforge.net/svn/trunk/rpms/hylafax/
> http://svn.rpmforge.net/svn/trunk/rpms/kannel/
> For example they contain an <name>.logrotate file which
> becomes /etc/logrotate.d/<name> or an <name>.init file which becomes the
> file /etc/init.d/<name>.

For small files it is easier to have them inline. For a few reasons:

  - It is easier to see the total picture if it is included
  - If you have more than one SPEC file of a package, they can be different
  - You can use macros to modify the content of the files (eg. paths)

Even for init-scripts having them inline is useful as you can use paths 
coming directly from macros. I guess the person packaging can decide.
