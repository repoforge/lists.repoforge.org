> Yes, the sendmail that comes with EL2 does not have milter support (and 
> therefor clamav-milter cannot be build against it).

Not true.
I've built clamav-milter against RHEL 2.1 with official Redhat sendmail 
8.11.6 since clamav 0.72 was released.
Milter support in sendmail 8.11 requires and extra library at link time 
which clamav did not handle properly until the 0.75.1 release (where my 
proposed patch for configure was included upstream).

As I said I don't actually use clamav-milter but I've used other milters 
with RHEL 2.1 including when sendmail was at 8.11.6 and they've worked fine.

> If you however replaced the official sendmail with a newer one, you can 
> rebuild that package yoursellf by adding rpmbuild --with-milter.
>
