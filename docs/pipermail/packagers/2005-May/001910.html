I have a question about stability and quality of the packaged software.
While toying around with opensync (the old multisync) I made some rpms
for it. But this software is not yet intended for broad usage. 

Are these rpms RPMForge worthy? 
-------------------------------
	
If you/we include it in normal RPMForge, people might get the impression
that we don't value quality.

I see a number of solutions:

1. put the string "EXPERIMENTAL:" in the package description and
summary.	

2. create an experimental or beta or ... repository (however I think
this kinda sucks from a user's perspective)

3. not building them, simple having the specfiles could be worth it. 

I think users expect:
- packages that simple work after adding {yum, smart, ...}
- quality packages
- no fuss 

So I would go for option 1. 

What do you guys think?
