Well also in this case there is also some use in still keeping them. 
People that do have MQSeries and want RPM packages can build them 
themselves, or at least use the SPEC file as something to start from.

Plus they can send us feedback and improvements so we can offer something 
that is maintained by those people that have a use for it.

So also in thise case I prefer to keep them, just for that.

There is no rule that a SPEC file in subversion needs to produce an RPM to 
be available. Plus I would like to also offer nosrc.rpm packages for those 
things we cannot legally offer.
