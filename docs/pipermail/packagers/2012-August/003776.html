Well, it sounds like you didn't add your ssh public key to github, so
that it can properly authenticate you as the owner of the repository.

Initially, you cloned the repository using the git:// protocol that has
no provisions for authentication and is usually employed for anonymous
read-only checkouts only.

When you tried to push, github couldn't authenticate you and therefore
denied you the access. Otherwise, anyone would have been able to push to
your repository.

Now you changed the protocol to git+ssh://, but apparently there is
something wrong about your public key settings. While this problem can
be definitively solved, I can offer you another quick workaround.

You might make use of the new https:// protocol that can perform secure
authentication by password instead of public key over HTTPS:

    git remote set-url origin https://github.com/PaulReiber/repoforge.github.com.git

Now try to push again, and if your git is recent enough, it should ask
you for github username and password. If they do match with what github
has in their database, it should allow you to push.
