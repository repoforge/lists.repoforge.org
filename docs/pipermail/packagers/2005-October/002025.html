On Sun, 9 Oct 2005, Dries Verachtert wrote:

> What about adding a tag like the following for stuff which isn't finished yet?
> 
> # Tag: workinprogress

Well, I use the Tag for tagging packages differently. So if you use '# 
Tag: test', the resulting package will have a release like:

	-1.2.el4.test.i386.rpm

instead of

	-1.2.el4.rf.i386.rpm

Which in effect will make sure the package is not added to a repository.
We can of course name it '# Repotag:' to make that more clear.

I prefer to sign off packages by builders and consider it 'in progress' if 
not everyone has signed it off. (or at least hasn't been signed off by at 
least a few builders/packagers)

	# Signoff: dag, dries, thias

Would mean that these 3 persons have blessed the SPEC file.
