Hi,

Now that we have cleaned up the RPMforge repository, we may want to verify 
that indeed no packages replace base packages on RHEL3, RHEL4, RHEL5 and 
RHEL6. So if anyone wants to do a check in a virtual machine, that would 
be appreciated.


Besides this, we do have another problem, which is that some packages 
dependending on packages that are now in RPMforge-extras are still in 
RPMforge. And we have to move these over as well. I would really like to 
run a script (repoclosure) every day/week that performs this check and 
report dependency problems to this list. Now that I have a new 
(silent) system I think this is feasible.


Also the perl packages that we provide that replace a module that ships 
with Red Hat's perl package still remain in the RPMforge repository. We 
may want to move those over to RPMforge-extras (or simply stop providing 
them).


Currently RPMforge-extras is lacking mostly the perl packages that used to 
be in RPMforge. I am slowly rebuilding and adding them, but still I wonder 
of we really want to do this. Originally we didn't, but as the set of perl 
packages grew, we added perl-packages replacing base packages. Do we still 
want this for RPMforge-extras ?


Expert opinions needed ;-)
