On Mon, 2011-01-31 at 22:35 +0000, Jesse Hathaway wrote:

> How is replacing X.Org any more invasive than for example replacing the http
> server on a webserver with regard to backporting.

X.Org is a major component of the system which normally runs with
elevated privileges all the time and accesses diverse low-level kernel
interfaces. 

There is a lot room for potential conflicts, instability and failures in
the scenario where latest kernel and X.Org are transplanted on top of a
significantly older userland without having an idea on how they fit
together (which seems to be the case).

Replacing httpd, with, say, nginx (or, even, installing them side by
side as is very often the case) is nowhere near as invasive, because the
web server does not actually need root privileges most of the time, and
in the case when it uses upper ports, it doesn't need them at all. 

Moreover, the web server never uses privileged kernel interfaces
directly. It is a self-contained piece of userland software which is
confined to its working environment and is pretty much kernel-agnostic.
