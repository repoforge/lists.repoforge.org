Very easily. Dag is not committing his SPEC bumps again. Last time we've
got to the point when I was trying to find out how exactly this is
happening, but got overloaded with other things and dumped the thread.

My overall conclusion so far is that we have to blame Subversion for
everything. Dag is doing his work / private / not ready for release
stuff in the same trunk as regular RF updates and therefore he can't
just commit at the top level. 

At the same time, branching in Subversion is so painful and useless that
it's just unfeasible to create a branch for private experiments... It
gonna take twice the space of the repository and then you can not merge
it cleanly without losing the history etc.

Well, let's see. I am still very slowly working on git migration in the
background, so one day this might actually happen. If it's not ditched
after more or less everyone agreed on something, just as Trac, of
course.
