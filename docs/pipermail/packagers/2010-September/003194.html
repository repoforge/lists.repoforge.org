On 08/31/2010 08:19 AM, Christoph Maser wrote:
>> I am convinced that if we move to a branch-by-default strategy, that RHEL4
>> and RHEL3 are disregarded. And as soon as we have RHEL6, packages for
>> RHEL5 will also slowly fade away. I don't like that at all.
>
> I see your point but on the other hand building a buildsystem wich can
> handle both (unifed and per target specs) is much harder then to say we
> always have a per target spec. A default setup could have a symlink to a
> unified spec in each target dir.

How about setup the buildsystem to always build for each arch/distro - 
and only use a different spec for a specific arch/dist if it exists. So, 
if there isnt a el3/<spec> just build the main spec for .el3 / .el4 / 
.el5 ( as an example... the system can be made a lot smarter )

The way I look at it - on the buildsystem side of things, it only needs 
to be done once. If doing it on a per spec basis, the work needs to be 
done many many times.
