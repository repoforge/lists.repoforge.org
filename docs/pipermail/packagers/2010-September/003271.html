Hi Ariel,

Thanks for your feedback. We have a mailinglist for discussing such 
problems.

 	http://lists.rpmforge.net/mailman/listinfo/packagers

RPMforge used to be very cautious regarding updating base packages (we 
would only allow it for leaf-packages like eg. lftp or rsync) where the 
impact is very low.

But over the years upstream would add new packages we offered, or mistakes 
made by packagers caused packages to replace base packages (perl packages 
are very common). The idea was that protect base and others allowed users 
to set their policy and we could provide a single repository to everyone's 
benefit.

For many months now it's clear we may want to split the repository in one 
that does not replace base packages, and one that may. This has its own 
problems, like some package (eg. inkscape) works fine with the base 
libraries, at some point it needs some additional package that requires a 
newer perl package. In this case inkscape would move from the base 
repository to the replace-base repository. People would stop receiving 
those updated packages...

Coming back to your specific case, yes this is an unfortunate problem 
which needs to be addressed :-/ It was introduced for cifs-utils, but it 
might well build against an older libtalloc.
