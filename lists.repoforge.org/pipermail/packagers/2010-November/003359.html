There are a few issues that I need more information about:

=> C++ build problems

One example is kobodeluxe-0.4.0:

window.cpp: In member function 'void window_t::center_token_fxp(int, int, const char*, signed char)':
window.cpp:313: error: invalid conversion from 'const char*' to 'char*'
make[2]: *** [window.o] Error 1
make[2]: Leaving directory `/dar/build/KoboDeluxe-0.4pre10/graphics'
make[1]: *** [all-recursive] Error 1
make[1]: Leaving directory `/dar/build/KoboDeluxe-0.4pre10'
make: *** [all] Error 2


=> patch --fuzz=0 problems (patches that work with RHEL5 and older, fail
    on RHEL6)

One example is subversion-1.6.13:

+ echo 'Patch #2 (subversion-0.20.1-deplibs.patch):'
Patch #2 (subversion-0.20.1-deplibs.patch):
+ /bin/cat /dar/rpms/subversion/subversion-0.20.1-deplibs.patch
+ /usr/bin/patch -s -p1 -b --suffix .deplibs --fuzz=0
1 out of 1 hunk FAILED -- saving rejects to file build/ac-macros/aprutil.m4.rej
error: Bad exit status from /dar/tmp/rpm-tmp.ZaZbfN (%prep)

Another is dia-0.97:

+ echo 'Patch #3 (dia-0.94-fallbacktoxpmicons.patch):'
Patch #3 (dia-0.94-fallbacktoxpmicons.patch):
+ /bin/cat /dar/rpms/dia/dia-0.94-fallbacktoxpmicons.patch
+ /usr/bin/patch -s -p1 -b --suffix .fallbacktoxpmicons --fuzz=0
2 out of 2 hunks FAILED -- saving rejects to file app/interface.c.rej
error: Bad exit status from /dar/tmp/rpm-tmp.L25DNM (%prep)


=> Some requirements no longer exist in RHEL6, eg. glib, imlib, gtk+ and
    others. If we want older applications to be available, we need those.


=> Some things have changed regarding the KDE/QT headers, the old tricks 
stopped working

One example is filelight-0.6.4:

checking for KDE... configure: error:
in the prefix, you've chosen, are no KDE headers installed. This will fail.
So, check this please and use another prefix!
