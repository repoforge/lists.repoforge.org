Hi,
as of RHEL6 we have a php>5.2 platform available. It arises questions to
me, that we can support phpmyadmin 3.x finally. So the question is: what
package naming convetion are we going to introduce? We have to continue
supporting 2.x branch for el{3,4,5} and for el6 we can introduce 3.x
branch. Some scenario is:
phpmyadmin el{3,4,5}
phpmyadmin3 el6

I personally don't like this way. If there's some better I'd like to
know. With el5.6 upcoming we are facing the similar issue, since el5.6
is going to include php53 alongside with php package.
