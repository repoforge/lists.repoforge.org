hello folks!

you may recall a recent flurry of email about the NetAddr-IP Perl module breaking SpamAssassin.  unfortunately, the current release (which we ship) still contains an insidious bug that, while it doesn't prevent SpamAssassin from running like the previous bug does, instead breaks the internal_networks/trusted_networks functionality, which is going to result in more spam getting through.

since the upstream maintainer hasn't indicated when he expects to be able to fix this bug, and seems to have stopped paying attention to the RT ticket (https://rt.cpan.org/Ticket/Display.html?id=62521), i've checked in a user-contributed fix for this issue.  as a general rule, i don't think that we should be in the business of fixing bugs in the software we package, but in this particular case:

* the bug is significant, and it may not be obvious to users
* a fix is already available

so, i think we should ship a patched NetAddr-IP, or if not, we should pull versions of NetAddr-IP > 4.033 from the repo.  otherwise someone is eventually going to figure out that our SpamAssassin is essentially broken out of the box, and is going to start complaining.

-shuff
