Hi,

I've noticed that apt won't upgrade my perl-HTML-Mason RPM from  
1.3101 to 1.32.  The problem is that it thinks 1.3101 is newer than  
1.32, since 3101 > 32.

My workaround was to put this in /etc/apt/preferences:
   Package: perl-HTML-Mason
   Pin: version 1.32*
   Pin-Priority: 1001
This causes apt to install perl-HTML-Mason-1.32 while printing a  
warning that is is being downgraded.

I believe, however, that the proper solution would be to increment  
the epoch number on perl-HTML-Mason.

Thanks,

Derek
