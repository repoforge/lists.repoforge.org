Dries,

If a committed SPEC file is not buildable, please do not increase the 
release.

If a committed SPEC file does not build on a single distribution (or a 
few of those) but have been build for other distributions (and do not 
require a rebuild for these), please do not increase the release.

Every change in the release-tag will force a rebuild on all other 
distributions. So everything that is cosmetic or distribution-specific 
(without impacting other distributions) do not require a change in 
release.
