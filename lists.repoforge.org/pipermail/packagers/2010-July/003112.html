Hey guys,

I would like to propose a few things:

  - A new repository tag, eg. 'rfx' for packages that replace upstream
    packages. Based on this tag, which is set by the Comment-header
    '# Tag: rfx' we could have these packages show up in the RPMforge
    Extras repository. A better name may be required, but keep it simple :)

    We could also decide to use a macro for the repotag so that we can make
    the repotag conditional (eg. use rfx for el5, but rf for all others)
    although that complexity may be unwanted ?

  - We could do the same as above for testing, and use the 'rft'
    distribution tag instead of the 'test' tag we currently use ?

  - I would also like to change the default buildroot for all SPEC files
    to:

 	%(mktemp -ud %{_tmppath}/%{name}-%{version}-%{release}-%{dist}-%{arch}-XXXXXX)

    which would already be very useful to me, but could be equally useful
    to others.

Any ideas ?
