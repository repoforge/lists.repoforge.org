Dear rpmforge packagers

i would like to split the nagios-nsca package in two parts (nagios-nsca
and nagios-nsca-client). Currently if you need send_nsca from rpmforge
you have to install nagios-nsca which needs package nagios as
dependency.
To get rid of this dependency i would like to make a subpackage
nagios-nsca-client wich contains the following files:

    /etc/nagios/send_nsca.cfg
    /usr/sbin/send_nsca


When these files are removed from the package nagios-nsca everyone
needing send_nsca would have to replace nagios-nsca by
nagios-nsca-client.
To void this we can put the files in both packages, wich makes them
conflicting.

Any comments? Is putting the same files in different packages ok?
