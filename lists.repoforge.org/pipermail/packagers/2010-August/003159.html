On 08/23/2010 09:54 AM, Dag Wieers wrote:

What sort of a git-repo layout were you think about ? Just importing the 
whole svn into git is easy, I've just run that  and you can get the git 
repo by :

git clone git://nazar.karan.org/testing/rpmforge

its a git-svn'd repo from svn/trunk

You'll see that something like this is fairly easy to work with, if you 
are working with a large set of packages - but not so much if its a 
single or a few packages one might be interested in, in which case it 
makes most sense to build the git repo's on a one-repo-per-package basis.
