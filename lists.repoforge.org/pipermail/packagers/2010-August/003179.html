On Mon, 30 Aug 2010, David Hrbáč wrote:

> Dne 29.8.2010 17:43, Christoph Maser napsal(a):
>> I would like to have the rpmforge spec format changed. I think lines 2
>> and 3 are not very useful. I don't see any use in line 2 "Authority" and
>> line 3 "Upstream" and would like to have them removed.
>> Especially for the upstream-line it sometimes takes a lot of work to
>> find the right address just to learn a couple of months later that it
>> was changed. I think the upstream project URL in the "URL: " field is
>> sufficient.
>
> OK, I agree. On the other hand I'd like to see something like
> "Source0-md5" used by PLD Linux.

I don't agree. The purpose of the upstream-address is that at some point 
in the future we can communicate with upstream. Either through a 
web-interface or by email. (If they are interested) But it's not at all 
mandatory.

The authority is currently our only way to know who is in charge of that 
package. Since there is no governance model, that's the only thing we 
have.

I do agree that having Source hashes would be useful, but we have no 
build-system that uses it. But if we have something that can use it, sure 
I'll add it immediately.
