On Aug 11, 2010, at 3:39 AM, Yury V. Zaytsev wrote:

> By the way, does anyone know is there a consistent Fedora/RH policy on
> Python modules with regards to PYC files? For instance, I am not quite
> sure that they are arch-indep.


hm, interesting.  Fedora says include everything:

http://fedoraproject.org/wiki/Packaging/Python#Byte_Compiled_Files

When installing python modules we include several different types of files.

    *.py source files because they are used when generating tracebacks
    *.pyc and *.pyo byte compiled files
          o python will try to create them at runtime if they don't exist which leads to spurious SELinux AVC denials in the logs
          o If the system administrator invokes python with -OO, .pyos will be created with no docstrings. This can break some programs.
    *.egg-info files or directories. If these are generated by the module's build scripts they must be included in the package because they might be needed by other applications and modules at runtime.
