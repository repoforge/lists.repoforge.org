> If it works with my current static build environments I could rebuild
> those static build environments again. But I do not want to rebuild my
> build environments for every build (that would be insane as I build a

>

Well mock does not re-install the chroot completely. If there is no
cache-file a fresh chroot is poulated and packed in a tgz. For
subsequent builds this tgz is unpacked and treated with yum update. This
can be quite fast on ramdisks. (Or really fast with LVM snapshots if
such a solution does exist)
For completely static environmens you can use --no-clean and
--no-cleanup-after. But this aproach leads to the chroot getting bigger
and bigger as more and more build dependencies get installed. Wich in
again leads to the scenario where you can forget a build dependency and
your rpm still builds fine since the requirements have been installed
before for a differnt build.

Yes it is slow, but it gives deterministic results.
