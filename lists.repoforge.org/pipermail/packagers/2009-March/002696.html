> I have a need for graphviz on el4 but the current rpmforge specfile is

> changes.

>
> Major changes:
> * More feature macros to enable/disable subpackages as appropriate
> * BR updates so that the language bindings are built correctly
> * Update to 2.22.0
> * Existing subpackages synchronized with upstream
> * Builds on all currently supported RHEL releases
>
> Note that the feature set enabled for Fedora (1-7 considered) and RHL
> 7-9 is entirely based on the upstream specfile since I do not have the
> necessary buildroots.
>
> With this patch the rpmforge package is closer to having feature parity
> with the packages that upstream makes available.
>
