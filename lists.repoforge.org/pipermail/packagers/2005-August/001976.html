On Fri, 19 Aug 2005, Wil Cooley wrote:

> I'm also using variables %{dist_tag} and %{vendor_tag} in my release.
> The docs mention %{repotag}; should I replace my %{vendor_tag} with
> this?  It doesn't seem to be defined in macros.rpmforge.  The docs also
> mention %{disttag} instead of %{dist_tag}, but I cannot find any of
> these tags actually in use.

The tools we use rewrite the SPEC file (in different ways) before doing 
the actual build. This means the SPEC file does not have any special dist 
or repo tags.

The things that happen during a rewrite:

 + Add Packager/Vendor depending on Authority/Builder
 + Add dist and repo tag to release
 + Trim changelog
 + Add subversion ID to changelog

The macros.rpmforge is currently not used. This is something I'm going to 
fix soon (ie. make the rpm-macros-rpmforge package usable).
