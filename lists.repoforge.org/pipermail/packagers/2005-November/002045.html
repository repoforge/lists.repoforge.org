RHEL, Fedora, SLES and OpenSuSE adopted JPackage's Java packaging
conventions.
JPackage provides all Java software, libraries, apps, as high quality RPMs,
supporting multiple JVM on same machine, compliance to the FHS, etc, with
YUM and APT.
Their problem is a no-source section where they provide only
nosrc.rpmpackages that has licencing issues, including the JVM,
forcing users to
build their installable RPMs just to start working with JPackage.

My proposal is to drop current RPMForge strategy to Java packaging, and
provide a built of these no-source packages. All the rest (no licence issues
software) can be grabed from JPackage's repo.

We'll have to only "rpmbuild" the SRPMs that are at their home
http://www.jpackage.org as the bottom right listing called non-free section.

I'm proposing to be the maintainer of these binary packages if you guys can
open a space in RPMForge server for them.

Comments ?

Regards,
Avi
