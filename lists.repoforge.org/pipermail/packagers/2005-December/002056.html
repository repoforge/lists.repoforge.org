I doubt it's intentional, so thanks for reporting the problems. Just to
make things clearer : freshrpms.net is just the subset of packages I focus
on in more detail, which are part of rpmforge. But there are two drawbacks
to this : Some breakage can occur from changes I make to the freshrpms.net
packages (the wxGTK update might be one example), and breakage might also
occur for older distributions since I usually only focus on the latest
Fedora Core release (FC4 currently).

I'll let Dag or Dries give you more details about the VLC and xine
problems.
