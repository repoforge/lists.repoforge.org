I don't think that upstream brings some value to packagers. Packager
always goes first to main website in case of package problem. On the
other hand I'd like to see different upstream info, which can be used
for automatic upstream version checker. I do use it, not included within
spec file, but within upstream list. One can check repository state vs
upstream state easily, e.g.:
