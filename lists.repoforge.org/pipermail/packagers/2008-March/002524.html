Due to the fact that apt.sw.be is *often* down during several hours,
isn't it a good idea to point the mirrorlist on another server than
apt.sw.be ?
Because it's good to have a mirror list that yum can use, but if the
list is on the more unstable machine, that doesn't make sense.
Why not creating a mirrorlist.rpmforge.net record that points to a more
stable machine and that contains only the mirror list ? So that machine
doesn't have to be a powerful one nor have much bandwidth .. just a
stable and always powered on machine  ;-) ...
