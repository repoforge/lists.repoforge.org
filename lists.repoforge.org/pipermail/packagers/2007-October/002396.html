Hi packagers

Nagios Conference 2007 just finished and it was a really nice
conference. Amongst other speakers Ton Von (project lead of
nagios-plugins) had a talk about state and roadmap of the project. In
this talk he mentioned that the next major release (version 1.5, no
timeplan for release yet) will in default configuration ship and install
Nagios::Plugin and all other perl-modules it depends on.

Personally i don't like that idea but in a short discussion with Ton and
the other core members it became clear they won't change it but they do
provide a configure flag to disable perl-module installation. I thought
i'd tell you before anyone gets the next (pre)release so you know what
is coming up.

My personal proposal for building nagios-plugins 1.5 would be disable
module-building and set a dependency on perl-Nagios-Plugin and package
that seperatly beforehand from CPAN. I already have a version of that
since i use actively ->
http://members.financial.com/~cmr/rpmforge/perl-Nagios-Plugin/

Have a nice weekend
