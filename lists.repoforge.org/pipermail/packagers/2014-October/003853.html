Dear all,

I'm experiencing problems with VLC while playing MPEG4 network
streams. VLC seems to be unable to open the stream. The stream is
opened using an SDP file from a video encoder, as it is published
using SAP announcements. However VLC gives the following errors when
trying to read the stream:

[0x7fb128c01e40] live555 demux error: Nothing to play for
rtsp:///tmp/stream_system_8_cam_81_live.sdp
[0x7fb128c01e40] ps demux error: cannot peek
[0x7fb128c01e40] mjpeg demux error: cannot peek
[0x7fb128c01e40] mpgv demux error: cannot peek
[0x7fb128c01e40] ps demux error: cannot peek
[0x7fb134000c90] main input error: no suitable demux module for
`file/:///tmp/stream_system_8_cam_81_live.sdp'

This applies to vlc-1.1.13-1.el6.rf.x86_64 currently present in the
repoforge repositories.

Long store short, I couldn't find the cause and in the end I
recompiled vlc from the source rpm in the repoforge repositories (the
exact same version). For some reason after recompiling vlc, it plays
the stream perfectly.

Firstly, what worries me is that I cannot explain why the recompiled
version does work out of the box, while the original version did not.
Secondly, it could be useful for other if the problem would be solved
in the repository as well.

How can I help to improve the vlc version in the repositories?
