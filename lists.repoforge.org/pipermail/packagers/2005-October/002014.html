On Fri, 30 Sep 2005, C.Lee Taylor wrote:

> 	I see that Matt and Dag have freedesktop and without_xorg spec
> options, but I don't really know what they are, I just copy them ... I'm sure
> they have something to do with RedHat and Fedora Core and integrate with multi
> build systems ...

These are conditional macros, used for building on distributions that does 
not have something that is an optional requirement.

In case of without_freedesktop, older distributions do not have 
freedesktop infrastructure. So we have to instruct the build to not use it 
for those. 

About without_xorg, I'm not sos sure this is necessary. The xorg packages 
include XFree86 provides iirc, so I wouldn't put them in there (just yet), 
but it doesn't hurt.

Normally, the default is what the most recent distribution (FC4) uses.
That's why often without_* is used instead of with_*

Also beware that in case something doesn't build for something older, 
you're not obliged to use these macros. Eg. in case something doesn't 
build on RH9 or lower, without_freedesktop is useless.
