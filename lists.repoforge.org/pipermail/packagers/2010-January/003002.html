Thanks to Karanbir I found how i could simply modify how mock builds the 
buildroot. One can just have a look at the /usr/bin/mock python code to 
find the defaults but just defining something else in the 
/etc/mock/'your.config.file.for.rpmforge'.cfg does the trick (so no need 
to rebuild the buildsys-build package) :

# using rpm-macros-rpmforge in the chroot
config_opts['chroot_setup_cmd'] = "install buildsys-build 
rpm-macros-rpmforge"

And of course mock is pointed to the corresponding repo to find the 
dependencies ;-)
