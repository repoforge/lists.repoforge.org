Often the YYYYMMDD scheme is used because there is no real version 
anywhere mentioned, except the date. I wouldn't mind to formally move to 
something we can agree on, even using the date as the version if we agree 
on that.

I don't think upstream will ever package hdapsd, so the problem we want to 
prevent in this case may never actually happen. But I prefer a proper 
solution by default.

ntfs-3g is different in the sense that they deliberately use the YYYY.DD 
convention as their version, and it is widely packaged as such. The 
maturity of the project definitely plays a role in what we decided to do, 
even if not formalized :-/
