hello folks!

so, RHEL5v5 just dropped, and it includes libevent-1.4.13, which seems not to be backwards compatible with RHEL5v4's libevent-1.1a.  it looks like the following packages will need a rebuild in order to be able to install on 5v5:

./arpd/arpd.spec:BuildRequires: libdnet-devel, libevent-devel, libpcap
./fragroute/fragroute.spec:BuildRequires: libdnet-devel, libpcap, libevent-devel
./honeyd/honeyd.spec:BuildRequires: libpcap, libdnet-devel, libdnsres-devel, libevent-devel
./libdnsres/libdnsres.spec:BuildRequires: libevent-devel
./memcached/memcached-1.1.spec:BuildRequires: libevent-devel
./memcached/memcached.spec:BuildRequires: libevent-devel
./nttlscan/nttlscan.spec:BuildRequires: libpcap, libdnet-devel, libevent-devel
./perl-Event-Lib/perl-Event-Lib.spec:BuildRequires: libevent-devel
./perl-Event-Lib/perl-Event-Lib.spec:Requires: libevent
./php-pecl-http/php-pecl-http.spec:BuildRequires: php, php-devel, curl-devel, libevent-devel, openssl-devel
./php-pecl-http/php-pecl-http.spec:Requires: php, curl, libevent, openssl
./sharedance/sharedance.spec:BuildRequires: libevent-devel
./tor/tor.spec:BuildRequires: libevent-devel, zlib-devel, automake, autoconf, openssl-devel
./trickle/trickle.spec:BuildRequires: libevent-devel
./vomit/vomit.spec:BuildRequires: libdnet-devel, libevent-devel, libpcap, automake >= 1.4
./vomit/vomit.spec:Requires: libdnet, libevent, libpcap

what fun!
