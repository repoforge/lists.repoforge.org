The recent upgrade to ddclient added SSL support, and the default 
preferences file has ssl=yes.

If ssl=yes ( or an ssl connection is specified from the command line ) 
their is a requirement for the IO::Socket::SSL perl library.

Apparently rpmbuild recognizes this as a conditional requirement and 
does not require the library for rpm purposes.

I recommend that either a Requires statement be added or the default 
preferences be changed to ssl=no with a NOTE added regarding the dependency.
