Attached is a diff to the current nagios.spec from svn. Aside from
bumping the version to 3.0.3 i modified the configure:

-    --localstatedir="%{_localstatedir}/log/nagios" \
+    --localstatedir="%{_localstatedir}/nagios" \

I think this is more correct then the current setup where
localstate-data and log-data are in one directory. (Maybe this should
even be %{_localstatedir}/run/nagios or %{_localstatedir}/lib/nagios ?)

To put the log files in a seperate location i introduced a vairable
%{logdir} and some sed-script to modifiy the config.

Some might object to this intrusive change. But i think it is ok to do
this here since nagios will run fine with the previous installed config
on upgrade. No config files are touched so its totally up to the user if
he wants to merge the changes from *.rpmnew files or not.

In any case i really would like to see nagios 3 being rolled out in
rpmforge and volunteer to support people who have problems with the
upgrade from 2 to 3.

Please comment
