> Well to each his own but I'd call that broken packaging and two wrongs

Actually that is done, at least for glibc, make, patch, tar etc etc. Try
"yum  groupinfo buildsys-build" on fedora. That is what is used to setup
the buildsystem in mock and see it contains also redhat-rpm-config.
Btw you also have macros for __gzip, __tar, __mkdirm  __make, __grep ...
this list goes on very long. Is relying on them also broken? I do expect
a build system to have those. And I also want to have a way to globally
define things for the build environment without repeating it in each and
every spec-files.

I don't see any problem here. If you call it broken fine, but by that
definition propably all larger RPM repositories produce exclusively
broken spec files.
