> hey Christoph - i recently stumbled across the following suggestion on
> the Fedora wiki:
>
> https://fedoraproject.org/wiki/PackagingDrafts/AutoProvidesAndRequiresFiltering
> #.25_docdir_filtering
>
> you may well have read this before, but the idea of explicitly
> removing %{_docdir} from the autoreq/prov scanner seems like a good
> idea, since if a module has example code it may be placed there.
>
> for many modules this will have no effect, since there's no Perl code
> in the docdir; for some modules this will eliminate bogus
> dependencies.  i would be surprised if adding this configuration were
> to break anything (that was not already broken), so i suggest to start
> adding this chunk of code to Perl module specs before the %description
> just as a matter of best practice:
>
> # we don't want to either provide or require anything from _docdir,
> per policy
> %filter_provides_in %{_docdir}
> %filter_requires_in %{_docdir}
>
> # actually set up the filtering
> %filter_setup
>
> thoughts?
> -steve
