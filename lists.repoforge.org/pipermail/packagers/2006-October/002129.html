I suddenly find fftw from apt.kde-redhat.org trying to override my  
version of fftw from RPMforge. The reason is that fftw version 3.1.1  
suddenly appears from kde-redhat with the package name "fftw", where  
version 3 everywhere else is called "fftw3". There is no simple  
upgrade from fftw version 2 to fftw version 3, because the API has  
changed.

So, all have to be aware that kde-redhat is not compatible with  
RPMforge at the moment. If you use kde-redhat, things will break if  
you upgrade fftw!!

  I hope the kde-redhat maintainers will change their package naming  
scheme ASAP so they use the package name "fftw3" for fftw version  
3.*. I am not sure why they include fftw in their distribution.  
Probably because some program in kde needs it. That program breaks  
with RPMforge, and soon other stuff will too.
