Yes! Actually, this is what I was referring to in my initial message and
the reply to Karanbir.

With GitHub you can create a git repository containing texts marked up
in any of those plain ASCII languages (asciidoc, markdown, textile etc.)
and it will make a wiki out of it. This wiki can be further edited via
the web interface or by checking out the repository and pushing changes
into it.

The same stands for the websites: it can generate HTML pages from a git
repository containing the plain text files and templates automatically.

I find this a brilliant idea and would support you by all means, but
personally I haven't used asciidoc (I use markdown instead) so I would
go for what I used :-) But if you are saying asciidoc, let it be
asciidoc. 

Hopefully it is not as complicated as that to learn.
