What i'm doing here is that my PPC builder *always* build from svn (i 
mean from a local and updated copy of the svn tree of course). So to me 
that means : no commit, no build.
If i have to test a package/spec, i test it on another machine and if 
that's ok i commit the spec to svn. And then it goes back downstream to 
the builder which process it.
Why not doing the same ? a sandbox/playground machine (or even the same, 
but a different process) on which some builds can be tested and once 
it's ok, commit in svn so that all svn followers know that there is an 
updated spec and so  a build to come ...
