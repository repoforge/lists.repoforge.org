I have a proposed patch to the scponly spec file. Its minor...

scponly supports a chrooted environement. To enable this, you must issue
an option to configure: --enable-chrooted-binary.

solution:

modify the configure arguments and add %{_sbindir}/scponlyc

There is a script called setup_chroot.sh in the doc dir which calls
config.h to aid in preparing the chrooted environment. I was considering
adding config.h into the doc dir, but I am not sure this is the best way
to address this. The config.h file is used in two places.

1) it obtains a list of binaries scponly should allow the client to
execute ( and copies them into the bin dir )
2) checks for the existence of config.h before it will run ... anything
really.

Any thoughts about the config.h?

Attached is a .diff against the current revision of the .spec file. I have
a buildlog if anyone wants to see it.
