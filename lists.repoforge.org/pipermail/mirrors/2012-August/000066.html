Chris,

You're not doing anything wrong - and this should have been explained to 
you a little more nicely.

The other administrator is correct, that the "img" folder is built by 
Apache web server, when Apache is installed.

Because most web servers run Apache, it is assumed that this folder is 
already there, so most repositories will not mirror that folder itself.  
Other common exclusions are "cgi" folders for custom scripting and such.

That aside, the easiest way to resolve your issue is to do a ONE TIME 
copy of the "img" folder to your NGIX web server.

Since the "img" folder never changes, only doing a one time copy will 
not be an issue.

I have attached the folder to this email.  You can simply unpack it, and 
then copy-and-paste it into your NGIX directory.
