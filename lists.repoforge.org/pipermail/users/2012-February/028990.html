I've put some updated SRPM patches and .spec files up on
www.github.com for tools I've been working with.

    https://github.com/nkadel/cpan2rpm-2.0.28-rpm - cpan2rpm .spec
files and patches for cpan2rpm for RHEL 5 and RHEL 6. cpan2rpm is an
old tool for building RPM's from CPAN perl modules, very useful for
building Repoforge candidates from CPAN modules.

    https://github.com/nkadel/daemontools-0.76-srpm - daemontools
.cpec file and patches for RHEL 5 and RHEL 6, with license information
updated. Uses initctl, not init scripts, to run daemontools. I
actively dislike this package, but people *do* keep asking for it.

    https://github.com/nkadel/samba-3.6.0-srpm - .Samba 3.6.0 .spec
file and patches for RHEL 5 and RHEL 6: Fedora 16 already has this If
we want it in repoforge, it should go in the "extras", not the
standard repos because it's a standard system package. But RHEL 5 and
6 are pretty seriously out of date on Samba and lack CIFS 2.x support.

    https://github.com/nkadel/subversion-1.7.2-srpm - Subversion 1.7.2
.spec file and patches. Suitable for adaptation to repoforge. Based on
Fedora 1.6.17 package, with the sort of patches and requirements of
sqlite-amalgamation for RHEL 5.
