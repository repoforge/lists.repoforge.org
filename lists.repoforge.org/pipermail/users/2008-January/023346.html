Mikel

i am all for using alternatives... but.
I  wonder if its really a good idea to make an alternatives entry for vi
in this package since it does not have any effect. Vi has no
alternatives and the user would have to manually replace /bin/vi by a
symlink. I would not call that system "standard install" afterwards.

The point behind vi not using alternatives is that the
alternatives-system  does not support different alternatives for
differnt users and the vendor wanted to make sure root gets /bin/vi when
he calls vi.

I think this idea is quite sane and we sould not advice users to
replace /bin/vi.
