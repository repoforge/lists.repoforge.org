On Fri, 1 May 2009, Phil Schaffner wrote:

> Did "yum install fuse-ntfs-3g" on CentOS 5.3 and it pulled in fuse as a 
> dependency but attempting to mount an NTFS partition resulted in errors:
>
> [root at hazardous ~]# emacs /etc/fstab
> [root at hazardous ~]# mount /XP
> FATAL: Module fuse not found.
> ntfs-3g-mount: fuse device is missing, try 'modprobe fuse' as root
> [root at hazardous ~]# modprobe fuse
> FATAL: Module fuse not found.
>
> Doing "yum install dkms-fuse" fixed the error, but shouldn't fuse pull in 
> dkms-fuse as a dependency if it is necessary?  (dkms was already installed 
> for other reasons.)

No, because fuse can be installed by using the CentOS Plus kernel. 
So we deliberately did not add any dependencies to dkms-fuse.

I know it sucks, but a google search should be helpful :)

CentOS 5.4 fuse will ship with the kernel so this issue is gone.
