In subversion-1.5.6-0.2.el5.rf, there is a directory
in /usr/share/doc/subversion-1.5.6/ there is a tools/ directory with a
number of subdirectories.  Some of the tools in this directory are
written in C.  The source for those tools is included, but in the
current RPM, they are not compiled.

It seems that these cannot be compiled simply from the command line.  In
order to build server-side/svn-populate-node-origins-index.c, I had to
grab the SRPM, install it, build the prep stage, then run configure and
make tools.

I would like to suggest that make tools be a part of the build process
for this RPM, so that these utilities are ready to use after
installation.
