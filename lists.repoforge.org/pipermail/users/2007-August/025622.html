The lzo2 package is calling for an executable stack, which breaks things 
like openvpn on Centos 5 and other systems where SELinux enforces against 
executable stacks.

See:

<http://bugs.debian.org/cgi-bin/bugreport.cgi?bug=420753>

A link is provided there for a patch against lzo-2.02:

<http://ftp.riken.go.jp/pub/Linux/gentoo/dev-libs/lzo/files/lzo-2.02-exec-stack.patch>
