> there appears to be a new perl-DBD-mysql package in the repo.
>
> I have yum-protectbase configured properly, I think (but I'm not sure).
> I certainly get "xxx packages excluded due to repo protections" and
> yum doesn't try to update other conflicting packages.
>
> Still, this package wants to update/replace the existing RHEL5
> perl-DBD-MySQL.
>
> I'm guessing protectbase does a string match on the package name, and
> the case difference is making it miss.  Meanwhile the rest of yum
> sees a newer "capability: perl(DBD::MySQL)" or someting like that
> and its update logic says it should replace the old package with the
> new one.  Obviously this isn't what we wanted if we installed
> protectbase.
>
> I'm not sure this is a protectbase bug or a repo bug, but it sure
> makes things inconvenient.
>
> If others concur with my guess, can this package be renamed to match
> the rhel package?

i've seen this be caused by "obsoletes" entries in the rpms.  you need to 
tweak the config for the yum plugin.  for the priorities plugin, it is:

check_obsoletes=1

maybe protectbase doesn't have an option for that, i'm not sure.
