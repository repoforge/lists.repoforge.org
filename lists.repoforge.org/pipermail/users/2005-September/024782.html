Perhaps? 

Let's take the rpm package for example. It's requires for libc are as 
follows:

libc.so.6
libc.so.6(GLIBC_2.0)
libc.so.6(GLIBC_2.1)
libc.so.6(GLIBC_2.2)
libc.so.6(GLIBC_2.3)
libc.so.6(GLIBC_2.3.4)

I currently have:

/lib/libc-2.3.5.so

But it still works!

Why?

Because rpm does not force a requires setup like this:

libc.so.6(GLIBC_2.3.4)

Or, heaven forbid, like this:

libc-2.3.5.so

The previous two setups would not allow an upgrade of glibc without a 
rebuild of everything that requires that specific version.  Now, 
because we know that the APIs usually are pretty robust across, at the 
very least, a micro version upgrade, we don't need to require such 
specific versions.

Essentially, requiring Python 2.4.1 is the same thing. If it weren't 
then I guess the lib directory should be this instead:

/usr/lib/python2.4.1

Am I wrong? Or were you agreeing with me in the first place? :P
