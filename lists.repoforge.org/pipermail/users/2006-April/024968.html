I think you are right.

I had a long list like this yesterday and managed to make it a lot
shorter by checking each package -- for some of them like libid3tag or
perl-MIME-tools for example will find that there are older version in
core or extras so doing a 'rpm -e' and a 'yum install' will get rid of
the rf package and install the fc5 version. 

However I couldn't do this for some packages -- for example
python-elementtree -- there is a older version available for fc5 from
core but since yum depends on it you can't rpm -e it and then do a yum
install (I know I could download it and do it manually using rpm...).

I also found some packages that I didn't seem to be using so I just
removed them so I now only have these fc4 packages installed:

 rpm -qa | grep fc4
 perl-Sys-Hostname-Long-1.4-1.2.fc4.rf
 perl-Term-ReadLine-Gnu-1.15-2.2.fc4.rf
 rpmforge-release-0.2-2.2.fc4.rf
 python-elementtree-1.2.6-5.2.fc4.rf
