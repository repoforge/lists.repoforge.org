Hi,

I recently posted a message proposing a fix in the clamav spec to make clamd 
work with SELinux in enforcing mode. There was no response to that post.

However, as of today a new version of clamav has been released. I would really 
like to see my fix integrated in rpmforge before the new clamav packages get 
built. Otherwise I have to continue to manually build the clamav packages 
myself.

Can somebody look at my improved spec file and (hopefully) check it in ? If 
you prefer a diff file, please let me know. I can arrange for that as well.

Thanks,

Geert

> I'm in the process of setting up Centos 5 with SELinux in enforcing mode.
> This causes problems with rpmforge's clamav build.
>
> Why ?
> Rpmforge's clamav-db and clamd pacakges use directory /var/clamav to store
> the virus definition databases. In the SELinux (targeted) policy as it
> comes with CentOS 5, this directory is of type "var_t" while clamd belongs
> to domain "clam_t". This domain is not allowed to access files of type
> "var_t".
>
> Solution:
> The CentOS 5 SELinux (targeted policy) specifies that directory
> /var/lib/clamav is of type "clamd_var_lib_t", which the "clamd_t" domain is
> allowed to access.
> So I have slightly modified the spec file to use /var/lib/clamav instead of
> /var/clamav where appropriate.
>
> I have verified this to work on CentOS 5 with SELinux in enforcing mode.
>
> The modified spec file is attached. If the rpmforge team agrees with the
> proposed solution, can you then please include this spec in your repo ?
