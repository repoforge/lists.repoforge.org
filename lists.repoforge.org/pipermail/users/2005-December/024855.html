Well, I used to have all i386 packages included in the x86_64 repository 
but that triggered another yum bug that Seth refuses to fix. (or rather 
denies it's a bug).

with the next bunch. Yum easily refuses to work in certain cases where it 
could simply continue to process the rest.

But I cannot start moving those i386 packages from the i386 to x86_64 
manually (my current design does not allow it, plus it would be a 
management nightmare since I cannot know what packages are libraries and 
what are not).

So I simply cannot help here. Your best bet is to only install x86_64 
packages if you require libraries from repositories that are not Red 
Hat's.
