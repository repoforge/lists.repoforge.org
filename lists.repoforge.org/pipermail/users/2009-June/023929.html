If neon is unpacked inside the subversion tree, it gets compiled in 
statically, similarly to what's currently done with sqlite.  I like this 
approach because then the user doesn't have to consider multiple 
versions of the same RPM (neon in this case) installed concurrently:  1 
for the OS dependencies and 1 for subversion, and the user doesn't have 
to consider the problems associated with updating a vendor supplied 
library like sqlite or neon to a newer version.  Even though it's 
supported, it just seems easier to maintain (for the user).
