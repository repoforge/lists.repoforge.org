I tried the new package now and found that it doesn't make much sense to 
distribute it via rpm. The Run script wants to compile the stuff, anyway. 
(I didn't notice this with the Unixbench package, I think, it just ran.) 
So, if you don't have make and gcc installed Unixbench 5 won't compile and 
won't run. And because of the optional rudimentary graphic benchmarking 
there's now a dependency for three graphics libraries, including 10 MB 
Mesa-GL. So, it rather looks one is better off with just downloading the 
source and running ./Run from there as there is not much difference.
When I yum removed unixbench it didn't remove the dependencies it had 
installed. Shouldn't it do that? At least for CentOS base packages I find 
that they always want to uninstall the dependencies. Sometimes that is 
unwanted as it could remove a lot of basic packages. Is that why the 
package didn't want to remove the three graphics libraries?
