There's a confusing problem uninstallation problem and a security issue when
installing nagios, and installing nagios-nrpe. Since nagios-nrpe by default
uses the same user as Nagios, and installs the same user by default,
attempts to uninstall the 'nagios' or 'nagios-nrpe' packages can delete the
nagios user and break the other package.

Nagios and NRPE are in fact separate services, and should really use
separate users. Moreover, there are Nagios configuration tools (like fruity
and its more recent version, lilac) which require write access to the Nagios
configuration but *should not* have access to the NRPE setups.

So I'd like to suggest that nagios-nrpe add a separate, nrpe user and in
fact keep its configuration files in /etc/nrpe/ instead of /etc/nagios/, for
configuration safety. Do folks agree with that approach? If so, I'll submit
altered .spec files and patches as necessary by next week. (I'm in the midst
of moving right now, and cleaning up old notes from my last consulting
work.)
