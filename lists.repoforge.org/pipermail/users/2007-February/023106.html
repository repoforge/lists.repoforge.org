I just realised, the yum complaint comes from the fact I was upgrading to 
the spamassassin 3.1.8-2 RPM supplied by SL (RHEL), as rpmforge doesn't 
supply this version.

Rightly so, spamassassin-tools is the one that generated this dependency 
complaint.

I receive daily yum check emails with outputs like:

-------------------- 
YUM - dag 
-------------------- 

spamassassin.x86_64                      3.1.8-2.el4            sl-errata    

and because I use mrepo then the rpmforge checks are done with the "sl-
errata" channel. It didn't occur to me that this was an SL upgrade and not 
an rpmforge upgrade.

Red Hat have upgraded to this version (from 3.0.6) to fix an SA 
vulnerability:

http://rhn.redhat.com/errata/RHSA-2007-0074.html

Would that fix also require rpmforge to upgrade?
