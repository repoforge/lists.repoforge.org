do *not* upgrade to 3.3.1 - this release contains bugs and incompatible 
changes!

even the default theme is now changed by default, and the call home 
functions are increased with an integrated rss feed on main.php. next to 
that, empty perfdata check results are not put on perfdata channel 
anymore so ohcp/oscp and various grapher addons won't work properly 
anymore. notification macros contain memory leaks and even the html 
files do not install properly with make install. for further 
information, read the nagios-* lists.

i'm keeping an unofficial fixed version using backported fixes from 
icinga, but i truly recommend staying with 3.2.3 or below.
see https://github.com/dnsmichi/nagios-fixed/commits/3.3.1-fixes but 
from reports the provided spec file doesn't work anymore, due to many 
undocumented changes.

but, as a matter of fact, the nagios devs should release 3.3.2 or better 
3.4.1 to mark the next stable releases, than wild hacking for rpms.
