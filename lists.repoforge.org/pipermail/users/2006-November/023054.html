I talked to Dag some time ago bout JPackage, proposing to distribute binary
JPackage RPMs, but the converstion went nowhere, and several RPMForge
Java-dependent software require a Dag-style Java packaging, at that time.

I think RPMForge should distribute built packages from the JPackage's
nosrc.rpm collection because this is the established standard. And not its
own style Java packages.

I don't know if he changed the RPMs. Anyway, the Java source code, as far as
I understood, will be available next year only. So in the mean time, I'll be
happy to see RPMForge distributing JPackage stuff.

Regards,
Avi
