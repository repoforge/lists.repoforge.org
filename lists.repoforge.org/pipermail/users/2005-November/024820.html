seem to be any problem in the el3 repository, I cannot check other
repositories, as I don't have them available).

from filelist.xml.gz and filelist2 is "sha1sum package" for each package
in filelist. The strange occurences of files having multiple sha1sums do
happen when there's more than one entry for the file in filelist.xml.gz:

[root at forge repodata]# zgrep -n cpuburn filelists.xml.gz | cut -d: -f1
374
973
1502

So could someone please rebuild the metadata on there?
