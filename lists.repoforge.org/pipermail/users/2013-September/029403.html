Hi everybody,

As you might discover the current state of the RepoForge is very unhappy. Maybe the project is at the end. We do not have updates for about six months notwithstanding we do have quite a lot of commits within the repository on GitHub.

We as developers/commiters do not have access to RepoForge build stack and the signing key. Dag has no time, we all understand this, but the community is in a bad need of updates. Especially the security ones.
Because of this situation I have prepared update repository to handle the updates outside the RepoForge scope. Packages are taken from RepoForge GitHub repository and signed with my personal key. I'm thinking about creating a new key for RepoForge project and resigning in the future.
