On 06/25/2009 12:46 PM, Philip Durbin wrote:
> Yes, that is correct; if I run "yum install libgeos.so.2 gdal" an older 
> version of geos (2.2.3) is successfully installed.  However, I have an 
> application (GeoDjango) that requires geos 3.0.0 or higher.  So it would 
> be great if both gdal and the latest version of geos available in 
> RPMforge (3.1.0) could be installed side by side.

Can RPMForge rebuild gdal so it's linked against geos 3.x?

On CentOS 5 I installed geos-3.1.0-1.el5.rf and then ran rpmbuild with 
the same gdal spec file that's already in RPMForge: 
http://dag.wieers.com/rpm/packages/gdal/gdal.spec

This allowed me to install the latest versions of geos and gdal 
available in RPMForge side by side:

gdal-1.4.4-1.el5.rf
geos-3.1.0-1.el5.rf
