Hmmm, I didn't really think about it, because I don't let RPMForge to
replace the base packages via priorities and protectbase anyway.

that it provides an xmms plug-in which is not normally built by Redhat,
because they don't provide xmms, so it's of no use.

I came to the conclusion that the real solution for this problem would
be to rewrite the spec, so that it builds the xmms plugin only against
the flac-devel package which is provided by the base repo.
