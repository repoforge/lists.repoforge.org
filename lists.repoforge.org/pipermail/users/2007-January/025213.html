Hi,
I just did an rpmbuild of scribus 1.3.3.6 using the spec file from Dag's
site after changing the version number:
http://dag.wieers.com/packages/scribus/scribus.spec

I noticed that the python scripting was not included in the resulting
package (using CentOS 4 x86_64). Doing a ./configure revealed that it
was not detecting python2.3 . I have checked the 1.3.3.3 packages for
el4 and fc3 both i386 and x86_64. The i386 packages have the scripting
included, the x86_64 packages do not.

Search of the scribus lists revealed the answer in this thread:
http://nashi.altmuehlnet.de/pipermail/scribus/2005-August/012689.html
which was to add the option "--with-extra-libs=/usr/lib64" to configure.

With this option added, it builds fine and includes the scripting. Sorry
I don't know how to add this to the spec file so that it only applies to
x86_64.

Thanks for the good work.
