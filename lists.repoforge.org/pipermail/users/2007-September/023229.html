There are other packages in rpmforge which for example need a recent version 
of kde or gnome. Normally this is ok if the spec file contains a nice 
buildrequirement which checks for the correct version so it fails immediately 
on older distributions (*). Dag also often maintains multiple spec files of 
different versions of the same program so he's able to offer rpms for a wide 
range of distributions.

(*) "do as i say, not as i do" if you know what i mean :-) I must confess i 
still got some spec files which don't contain such a buildreq.
