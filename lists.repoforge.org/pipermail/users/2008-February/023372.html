I already sent this to the users-list which was obviously the wrong place.

ruby-shadow is currently only available for el5, so i investigated the 
issue and came up with a solution. It isn't perfect, but it works for 
rhel/centos/fedora.

I introduced a macro %has_ruby_abi that is set on rhel/centos/fedora >= 
5 so the specfile can decide wether to
Require: ruby(abi) = 1.8
or
Require: ruby >= 1.8.1

additionally the Makefile is "patched" after the build, because the 
install-phase broke on centos4

Still TODO:
- the macro-definition works, but is somewhat ugly/nonportable. Maybe 
there's a better approach?
- the install-phase shouldn't break - patching the Makefile between 
%build and %install isn't a good solution
