On Feb 21, 2008, at 12:50, Nils Breunese (Lemonbit) wrote:

> Looking at http://dag.wieers.com/rpm/packages/perl-MIME-tools/  
> there are non-test versions, yes.

True.  So, to elaborate, through yum updates I had:

   perl-MIME-tools-5.425-1.fc7.rf

on my system.  RPM tells me that it requires perl(File::Temp), but I  
don't see that requirement in the current .spec, which confused me.   
Because perl on f8 includes File::Temp now, without a Provides/ 
Obsoletes for perl-File-Temp, I was having trouble doing an upgrade,  
because perl-MIME-tools wasn't seeing its dependency satisfied.

So, I wound up rebuilding the same version of perl-MIME-tools (from  
the -test SRPM) as I had installed, building a copy of perl with a  
Provides/Obsoletes for perl-File-Temp (patch sent in), and packing  
them into a transaction together to get over that hurdle.  I think I  
should be able to downgrade to the release version of perl from there.

So, I think my immediate problem is solved, but I'm still uncertain  
about how the version that yum updates gave me corresponds to a -test  
version off of the website.  I thought the answer I saw in the  
archives is that the real current SRPM is in SVN, which made some  
sense on the versions not lining up, but then the question still  
remains how to get to that SRPM.
