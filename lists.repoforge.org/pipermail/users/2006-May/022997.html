Well, I'm not providing patched kernels and I probably never will :)

What is possible however is to provide kernel modules. I used to provide 
kernel modules for 2.4 kernels, but stopped doing it because it is rather 
time-consuming and I hoped someone could take over kernel module 
development. Hence, nobody stepped forward :)

The feature to see what process is using the most disk-io however is very 
useful and in my opinion so useful it should be part of the standard 
kernel. I could create a few dstat modules that could make use of it when 
doing performance analysis.
