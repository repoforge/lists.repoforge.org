Size is not an issue at all. The point is that the dependencies are not
ok right now (httpd missing), and another one will be added (php) in the
future. Also the development directs towards a separate GUI project.
(nagios + ndo + database + php-gui). Another point is distributed
installs. If you use nagios in an agent fashion with a nagios daemon on
each host you certainly don't want to have httpd and php installed
everywhere.
