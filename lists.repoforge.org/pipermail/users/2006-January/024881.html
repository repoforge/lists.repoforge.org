Raman Gupta wrote:
> Hello all,
> 
> The TinyCa in the repository does not work with openssl 0.9.7f which is
> the current version of openssl on Fedora Core 4.
> 
> The TinyCA web page [1] indicates that TinyCA has been deprecated in
> favor of TinyCA2, which is documented to work with later versions of
> openssl.
>
> [1] http://tinyca.sm-zone.net/

I have built the source RPM for tinyca2 provided at the URL above for
Fedora Core 4 with the following two changes:

1) Changed dependency on perl-MIME-Base64 to perl-MIME-tools

2) Added /usr/share/applications/tinyca2.desktop to the %files section.

The patch is below.

It would be nice to get the tinyca2 RPM included in the repository
because of the openssl issue with tinyca.

Cheers,
Raman

--- tinyca2.spec        2005-10-22 10:37:09.000000000 -0400
+++ tinyca2.spec.new    2006-01-18 00:20:39.000000000 -0500
@@ -16,7 +16,7 @@
 URL:        http://tinyca.sm-zone.net/
 Group:      Productivity/Networking/Security
 License:          GPL
-Requires:      perl perl-Gtk2 perl-MIME-Base64
+Requires:      perl perl-Gtk2 perl-MIME-tools
 Packager:      Stephan Martin <sm at sm-zone.net>
 Version:    0.7.1
 Release:    0
@@ -91,6 +91,7 @@
 %dir %{_datadir}/TinyCA2
 %{bindir}/tinyca2
 %{_datadir}/TinyCA2/*
+/usr/share/applications/tinyca2.desktop
 %if %suse_version > 820
 %{_datadir}/applications/*
 %endif
