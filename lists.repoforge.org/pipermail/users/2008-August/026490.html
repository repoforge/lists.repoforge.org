 Greetings,

 Just did a yum upgrade, got new awstats and:

/usr/bin/awstats_buildstaticpages.pl: /usr/bin/perl^M: bad interpreter: No such file or directory

 I double checked and it's indeed using 8859-1 as the charset instead of unicode. I
fixed it on my system with:

perl -i -pe 's,\015,,g' /usr/bin/awstats_*

and had a successful run/update afterward. A couple of other files that might be problematic:


[root at darwin]# file `rpm -q --list awstats` | grep 8859 | grep -v txt
/var/www/awstats/lib/browsers_phone.pm:                    ISO-8859 English text, with CRLF line terminators
/var/www/awstats/lib/search_engines.pm:                    ISO-8859 English text, with very long lines, with CRLF line terminators

Can somone please have a look? Thanks.
