> Here's an updated spec file that restores a sample httpd config file.

> Trac project, and mod_python is the recommended method anyway, it

Here's yet another Trac spec. It updates to Trac 0.11.1 (released today) and 
restores the egg info. It also creates /var/trac (in the vein of /var/www), as 
the sample apache config will throw errors if it does not exist. I'm not sure 
if this is appropriate or not, please let me know.

It turns out that removal of egg.info prevents setuptools from being able to 
load the package. See this pastebin, reported in #trac by someone who ran a 
yum update and got our 0.11-1 package without the egg.info:

http://paste.lisp.org/display/64905
