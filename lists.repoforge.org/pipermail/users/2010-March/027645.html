Hi Rainer,

A sysconfig config file is always used to override shell variables that 
influence the initscript. In this case there is only one variable that can 
be defined: the mrepo config-file.

I just looked at the initscript and noticed that unfortunately this 
variable is called YAMCONFIG, a relic from the days mrepo was called yam.

So I will fix this in subversion for a future release. But you can 
override the default location of the mrepo configuration file (which is 
/etc/mrepo.conf).

But that file is not required for the initscript to work, so if you are 
not changing the location of the mrepo.conf file, there is no need to 
create the sysconfig entry.
