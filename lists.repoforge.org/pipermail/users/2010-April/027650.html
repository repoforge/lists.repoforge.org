On Thu, 1 Apr 2010, Paul Heinlein wrote:

> I just upgraded a test server (CentOS 5.4 x86_64, no selinux) to
>
>  * clamav-db-0.96-1.el5.rf.x86_64
>  * clamav-0.96-1.el5.rf.x86_64
>  * clamd-0.96-1.el5.rf.x86_64
>  * clamav-milter-0.96-1.el5.rf.x86_64
>
> /etc/init.d/clamd now takes *much* longer to start than the 0.95.x 
> versions did. clamav-milter will work for a while, then report "No 
> reply from clamd."
>
> Is anyone else seeing this? Did I miss some crucial 0.95 -> 0.96 
> configuration item(s)?

In later troubleshooting, I saw the same sort of delays when using 
clamscan.

I ended up fixing the issue by

  0) stopping all clam-ish processes
  1) tarring up /var/clamav (just in case...)
  2) removing ALL files from /var/clamav
  3) downloading main.cvd and daily.cvd
  4) making sure permission were correct on the *.cvd files
  5) restarting all things clam

It's quite snappy once again.
