re: https://bugzilla.redhat.com/show_bug.cgi?id=570321

The bucardo postgresql replication package is currently packaged by EPEL 
but for EL5 it requires a version of perl-DBD-Pg that's newer than is 
provided by base+updates.  (I'm using it successfully with the rpmforge 
version.)

The EPEL folks don't want to update perl-DBD-Pg inside their repo and 
don't want packages that can't satisfy with base+updates+epel 
(presumably for EL6 this won't be a problem).

The current SRPM has one mis-named 'Requires:' for EL5, but aside from 
that it's a maintained and working package.  Does this sound like a 
package rpmforge could provide a home for?
