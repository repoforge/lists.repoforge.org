There are differences between syslog-ng and rsyslog, some features that favour
syslog-ng and some that favour rsyslog, however because rsyslog will be pulled
into Fedora 8, then Red Hat likely sees this as the future syslog server for
it's enterprise platform.

I'm on the rsyslog mailing list and the author does alot of work on it very
regularly, with a vision to it being the best syslog server available.

If you're using RHEL-based releases for the majority of your setups, then it
seems to me for the long-run, it's a wiser choice to get the rsyslog
infrastructure in place now instead of spending time on syslog-ng.
