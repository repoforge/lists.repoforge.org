Because munin-1.4 is out, and it needs a newer rrdtool than the ones 
available from EPEL or RPMForge. We have some customers asking for 
munin-1.4 on RHEL4.

I built the deps myself btw, http://users.linpro.no/ingvar/munin/1.4/ , 
but getting a newer rrdtool into RPMForge will give better general 
availability.

Another small matter about the rrdtool specfile that had to be changed 
before it worked in mock: It's very seldom (probably never) it's 
possible to run rpm from within a mock changeroot. So the following does 
not work:

Requires: perl >= %(rpm -q --qf '%%{epoch}:%%{version}' perl)

I don't know if RPMForge generally require packages to build in mock, 
but it's generally a Good Thing :-)

Also, on el4, the xorg Type1 fonts is in the fonts-xorg-base, so if el4 
should ever be supported, this should be moved to the el4 conditional 
Requires.
